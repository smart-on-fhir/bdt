Optional customTokenClaims

customTokenClaims: AuthTokenClaims
  • Defined in src/lib/Config.ts:64

Custom values to be merged with the authentication token claims. NOTE that the following cannot be overridden:

  • iss (equals the clientId)
  • sub (equals the clientId)
  • aud (equals the tokenUrl)
  • jti random value generated at runtime

Optional customTokenHeaders

customTokenHeaders: AuthTokenHeader
  • Defined in src/lib/Config.ts:75

Custom properties to be merged with the authentication token header before signing it. NOTE that the following cannot be overridden:

  • typ (equals "JWT")
  • alg (@see tokenSignAlgorithm below)
  • kty (equals the private key kty)
  • jku (equals the current jwks_url if any)

Optional tokenExpiresIn

tokenExpiresIn: string | number
  • Defined in src/lib/Config.ts:97

Expressed in seconds or a string describing a time span (Eg: 60, "2 days", "10h", "7d". A numeric value is interpreted as a seconds count. If you use a string be sure you provide the time units (days, hours, etc), otherwise milliseconds unit is used by default ("120" is equal to "120ms"). If not provided, we will use "5m" as default.

type

type: "backend-services" | "client-credentials" | "none"
  • Defined in src/lib/Config.ts:25

Can be:

  • backend-services (default) all tests will be executed.
  • client-credentials - uses client_id and client_secret. Most of the authorization tests will be skipped.
  • none - no authorization will be performed and all the authorization tests will be skipped.
